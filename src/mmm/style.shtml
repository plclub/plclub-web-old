Style Guide

We would like submissions to adhere to the following guidelines in order to assist anyone who may be interested in looking at them.

  1. The formalization of the definition of F<: should be separated from the formalization of its metatheory. For example, the definitions of the subtyping relation, typing relation, and operational semantics should appear together rather than being separated by lemmas and theorems.
  2. Expanding on the previous point, code that is needed regardless of the particular object language being formalized should be separated from the code that is specific to F<:.
  3. Any definitions, lemmas, theorems that correspond to ones from the challenge description, including the paper proofs, should be commented as such. These comments should also include (brief) justifications as to why the correspondence holds and what, if anything, had to be made explicit in going from the definitions to paper to the definitions in the formal environment.
  4. We encourage developers to put up their own web pages, which we can link to, that provide code and describe their submissions.
  5. As a convenience, we would appreciate it if developers could include appropriate line counts with their submissions; see the submissions page for exact counts we would like.

Here, "separate" may mean splitting code into multiple files or using comments to mark section boundaries.
